### <27> 자스에서 `var arr = [];` 이 코드가 무엇을 의미할까?

```
  --------------------------------------------
  arr 변수 선언하기,
  Array 객체 생성한 후 Array 객체의 메위주를 arr 변수에 저장하기 
  이때 Array 객체 안에는 배열변수 0개이다. 즉, 배열 변수는 없다. 추후에 저장용으로 생성.
  다량의 데이터를 저장할 목적으로 Array 객체를 생성해 놓은 것이다.
  --------------------------------------------
  위 코드는 아래와 같은 코드 이다.
  var arr = new Array();
```

> <b>변수에 데이터를 저장하는 이유? 재사용 때문.</b>

### <28> 자스에서 `var arr = [1,3,5];` 이 코드가 무엇을 의미할까?

```
  --------------------------------------------
  <1> arr 변수 선언하기,
  <2> Array 객체 생성한 후 Array 객체의 메위주를 arr 변수에 저장하기 
      이때 Array 객체 안에는 배열변수 3개가 있다. 
      3개의 배열변수 안에는 1, 3, 5 가 저장되어 있다.
  --------------------------------------------
  각 배열 변수를 호출하는 방법은      arr[0]  arr[1]  arr[2]      이다. 
  대괄호 안에 인덱스 번호를 쓸수 있으므로 
  반복문으로 돌리면서 이 인덱스번호를 바꾸어 접근할 수 있다.
```

### <29> 자스에서 `var xxx = getDdd(3);` 이 코드가 무엇을 의미할까?

```
  --------------------------------------------
  <1> xxx 변수 선언하기,
  --------------------------------------------
  <2> getDdd 라는 이름의 함수를 getDdd(3) 식으로 호출하기.
      getDdd 함수호출하면서 3이란 데이터를 넘겨준것이다.
  --------------------------------------------
  <3> getDdd 라는 이름의 함수를 호출하여 리턴받은 데이터를 xxx 변수에 저장하기.  
      그 함수는 function getDdd( 매개변수명 ){~; return 데이터;} 의 형식을 가지고 있을 것이다.
  --------------------------------------------
  <주의> 함수를 호출한 지점이 `=` 오른쪽이다. `=` 오른쪽은 데이터가 나와야 하는 곳이므로,
          이 함수는 호출하면 반드시 데이터를 리턴해야 할 것이다.
  <참고> 관용적으로 get 으로 시작하는 함수는 리턴값이 있다.
  <참고> 관용적으로 set 으로 시작하는 함수는 리턴값이 없다. 
```

### <30> 자스에서 `var xxx = new Date();` 이 코드가 무엇을 의미할까?

```
  <2> Date 객체 생성한 후 Date 객체의 메위주를 xxx 변수에 저장하기. 
      이때 Date 객체는 현재 년월일시분초를 관리한다.
      현재란 지금 이 시점의 년월일시분초를 말한다. 즉, 흘러가는 년월일시분초를 말한다.  
  --------------------------------------------
  객체 생성 후 아마   xxx.메소드()    형식으로 Date 객체의 메소드를 호출할 것이다. 
  객체 생성의 목적은 메소드 호출이다.
```

```
  --------------------------------------------

        var xxx = new Date(1999, 12-1, 25);
  --------------------------------------------
  <1> xxx 변수 선언하기,
  --------------------------------------------
  <2> Date 객체 생성한 후 Date 객체의 메위주를 xxx 변수에 저장하기. 
      이때 Date 객체는 1999년 12월 25일 0시 0분 0초를 관리하고 있다.

  <요령> 월 셋팅 헷갈리므로 아래 처럼 하면 덜 헷갈린다.
```

### <33> 자스에서 `var xxx = function(a){~};` 이 코드가 무엇을 의미할까?

```
  --------------------------------------------
  <1> xxx 변수 선언하기,
  --------------------------------------------
  <2> 익명함수 function(매개변수){실행구문} 를 xxx 변수에 저장하기. 
  --------------------------------------------
      --------------------------------------------
      자스는 익명함수. 즉, 이름없는 함수도 데이터 취급을 하므로 변수에 저장이 가능하다.
      --------------------------------------------
      익명함수도 함수이므로 호출이 목적이다. 
      --------------------------------------------
      익명함수를 변수에 저장하므로 이동하기 쉽고 필요시 익명함수를 호출할 수 있다.
      --------------------------------------------
      변수에 저장된 익명함수 호출 방법은 아래와 같다.  
          -----------------------
          변수명( [데이터] )  // 데이터는 생략 가능.
          -----------------------
  --------------------------------------------
  <3> xxx 변수안의 익명함수 호출방법은 아래와 같다. 
  --------------------------------------------
      xxx(데이터)
```

> 이름을 주지 않은 함수 = `익명함수`

> 자스에서는 익명함수도 데이터 취급을 해서 변수안에 집어 넣을 수 있다.

### <34> HTML 파일 안에서 `<script src="common_func.js"></script>` 이 코드가 무엇을 의미할까?

```
  HTML 파일의 head 태그 안에 script 태그 를 생성하고, 
  common_func.js 안에 있는 공용함수를 사용하기위해, 
  수입할 html 파일이 있는 같은 폴더에 common_func.js 파일을 위치시키고 이 파일을 불러온다.

  --------------------------------------------
  common_func.js 파일안의 자바스크립트 소스를 복사해서 현 script 태그 사이에 삽입하라.
  --------------------------------------------
  그러므로 <script src="common_func.js"></script> 를 가지고 있는 html 파일안에 
  common_func.js 파일안의 자바스크립트 소스가 있는 것과 동일하다.
  --------------------------------------------
  src="common_func.js" 의미는 현재 이 html 파일의 저장 폴더와 동일한 폴더에 common_func.js 파일도 존재한다는 의미이다. 
  즉, JS 파일 이름만 나온다면 동일한 폴더에 들어 있다 라는 의미이다.
```
