#### # `사용자 정의 객체` 이어서...

```javascript
function Sungjuk(s_no, s_name, kor, eng, mat){  // 과목수와 상관없이 하려면 Array 사용.
    // ---------------------
    // 속성변수 선언
    // ---------------------
    this.s_no = s_no;
    this.s_name = s_name;
    this.kor = kor;
    this.eng = eng;
    this.mat = mat;

    // ---------------------
    // 메소드 선언
    // ---------------------
    this.getS_no = function( ) {
        return this.s_no;
    }
    this.getS_name = function( ) {
        return this.s_name;
    }
    this.getKor = function( ) {
        return this.kor;
    }
    this.getEng = function( ) {
        return this.eng;
    }
    this.getMat = function( ) {
        return this.mat;
    }
    this.getTot = function( ) {
        return this.kor + this.eng + this.mat;
    }
    this.getAvg = function( ) {
        return this.getTot()/3;
    }
}

  // 함수명이 대문자이면 대부분 생성자 함수.  
  // 객체 생성의 목적은 메소드 호출.  
  // 코딩 형태가 변수선언이라도 익명함수가 들어있으면 메소드이다.  
  // 익명함수가들어있는 변수는 메소드.     
```

### <24> 자바스크립트 [일회성 사용자 정의 객체]에 대해 설명한다면? 

일회성 사용자 정의 객체는 코딩상 재사용이 불가능하다.  

```
● [생성자함수] 없이 만들어지는 객체를 말한다.
● 일회만 사용할 목적으로 만들어지는 객체이기도 하다.  

**************************
[일회성 사용자 정의 객체] 생성 코딩 형식
**************************
  ------------------------------------------
  var 변수 = {
      속성변수명 : 데이터,
      메소드명 : function([매변2]){
          실행코드;
          [return [데이터];]
      }
  }
  ------------------------------------------
    속성변수는 0개 이상 선언가능
    메소드는 0개 이상 선언가능.
    실무적으로 본다면, 속성변수나 메소드는 1개이상 선언해야 한다.  
    실무적으로 [일회성 사용자 정의 객체] 안에 메소드는 잘 나오지 않는다. 속성변수만 사용하려고 씀. 
    주로 여러 데이터  한속에 모아 보관하고 사용할 목적으로 사용한다.  
  ------------------------------------------

**************************
[일회성 사용자 정의 객체] 의 메소드/속성변수
**************************
    변수.속성변수
    변수.메소드명(~)
```

- [일회성 사용자 정의 객체] 생성코딩

```javascript
// 실무적 코딩  
*************************************************
다량의 데이터를 저장해서 관리하는 방법
*************************************************
-------------------
방법1> 데이터마다 일일이 변수 다 만들어 저장한다.  (원시적 방법)
-------------------
    var kor = 91;
    var eng = 81;
    var mat = 71;
    var che = 61;
    var phy = 51;
    alert(kor)
    ------------------------------------------------------------
    변수에서 데이터를 꺼내는 방법은 변수명을 일일이 코딩하면 된다.
    다른 프로그램 쪽으로 데이터를 전달하려면 모두 전달해 주면 된다.
    그럼 받는쪽도 n개의 변수로 받아야 한다.(=비효율적)
    ------------------------------------------------------------
-------------------
방법2> Array 객체에 저장한다.
-------------------
    var jumsu = [91,81,71,61,51];
    alert(jumsu[0])
    ------------------------------------------------------------
    Array 객체에서 데이터 꺼내는 방법은 반복문 사용하여 jumsu[i] 을 코딩하면 된다. 
    다른 프로그램 쪽으로 데이터 전달하러면 Array 객체의 메위주만 전달해 주면 된다.  
    치명적 단점. 데이터가 무슨 데이터인지 알기가 힘들다.
    Array 객체에 데이터를 저장할 땐 단순한 일방적인 데이터를 저장한다. 
    예>1000개의 사과 무게를 저장할 때
        var appleWeight = [200,210,190,~~~]
-------------------
방법3> [일회성 사용자 정의 객체]에 저장한다.
-------------------
    var jumsu = {kor:91,eng:81,mat:71,che:61,phy:51};
    alert(jumsu.kor)
    ------------------------------------------------------------
    Array 객체에서 데이터 꺼내는 방법은 jumsu.kor 의 형식으로 코딩하면 된다. 
    다른 프로그램 쪽으로 데이터 전달하려면 [일회성 사용자 정의 객체]의 메위주만 전달해 주면된다.
    장점. 데이터가 무슨 데이터인지 알 수가 있어 꺼내기 쉽다.  
    단점. 반복문 사용이 힘들다.
    ------------------------------------------------------------

-------------------

일반 array 사용하면 변수명이 획일적이기 때문에 
[일회성 사용자 정의 객체]를 사용.  

과목명이 다 다르기 때문에 [일회성 사용자 정의 객체]를 사용.

같은 종류의 데이터를 쓴다면 array가 더 적합하다. ex) 사과의 무게.
```
