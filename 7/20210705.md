### <16> [입력 양식] 관련 태그의 종류는? (입.)

```
●입력 양식 관련 태그는 [데이터]를 [입력] 또는 [선택]할 수 있는 태그 또는 그와 관련된 태그를 말한다.
●입력 양식 관련 태그에 입력 또는 선택한 데이터는 서버의 웹프로그래밍 언어로 전달되고 최종 DB=오라클(Oracle) 에 저장된다.
-----------------------------
●입력 양식 관련 태그의 종류
-----------------------------
  ---------------------------
  form 태그
  ---------------------------
    ▶여러 [입력양식]을 1개의 그룹으로 묶는 태그다. 마치 택배상자와 비슷한 역할을 한다.
    ▶서버의 URL 주소, 데이터 전송방법, 등을 설정할 수 있다.
    ▶실질적인 입력양식은 아니나, 중요한 태그이다.
    -----------------
    ▶form 태그의 속성
    -----------------
      name   -> form 태그의 이름을 설정하는 속성명. form 태그가 여러개 있을 경우 구별하기위해 주는 속성이다.
      action -> form 태그 안의 입력양식에 입력(선택)된 데이터를 보낼 [서버 쪽 URL 주소]를 설정하는 속성이다.
      method -> 입력(선택) 데이터를 서버로 보내는 방법을 설정

                <예> method="get"  => 입력(선택) 데이터가 URL 뒤에 붙어 전송.
                                      보안성이 낮은 데이터 전송 시 사용.
                                      이 방식으로 데이터를 전송할 때 URL 주소에 전송 데이터를 삽입만 하면 되므로 코딩이 줄어든다.
                                      즉, 입력양식이 필요가 없다.
                                      method 속성 설정이 생략되면 method="get" 가 디폴트(default)이다.

                <예> method="post" => 입력(선택) 데이터가 안보이게 전송.
                                      보안성이 높은 데이터 전송 시 사용.
                                      이 방식으로 데이터를 전송할 때 입력양식에 데이터를 삽입해야 되므로 코딩이 늘어난다.
                                      즉, 입력양식이 꼭 필요하다.

      enctype -> 전송될 데이터의 형식을 설정. 문자를 보낼 때 또는 파일을 보낼 때 이 설정을 해야한다.
                 만약 입력 양식중에 파일업로드가 있으면 다음처럼 enctype 속성값을 준다.
                 -------------------------------
                 enctype="multipart/form-data"
                 -------------------------------
                 <주의> enctype="multipart/form-data" 를 생략하면 파일이 전송되지 않고 파일의 이름만 전송된다.
                 <주의> 파일 업로드가 없으면 enctype="multipart/form-data" 를 생략하면 된다.
      target -> form 태그 안의 입력양식에 입력(선택)된 데이터를 보낼 때 어느 화면을 이용해 보낼지 설정한다.
                target 속성이 생략되면 form 태그가 있는 현재 화면을 이용하여 웹서버로 전송한다.

  ---------------------------
  input
  ---------------------------
    ▶데이터를 입력 또는 선택(=체크)할 수 있는 [입력양식]을 출력하는 태그다.  
    -------------------
    ▶input 태그의 속성.
    -------------------
      type="text"     -> 한줄의 [문자열]을 입력받는 입력양식을 설정한다.
      type="password" -> 한줄의 [암호문자열]을 입력받는 입력양식을 설정한다.
      type="checkbox" -> [체크박스] 입력 양식 설정한다. 다중체크 가능.
                         얘만 등장하면 관련된 모든 프로그래밍은 복잡해진다. DB의 테이블 구조도 복잡해 진다.
                         우리 개발자의 최대의 적이다.
      type="radio"    -> [라디오버튼] 입력양식 설정한다. 다중 체크 불가능.   
      type="file"     -> [파일박스] 양식 설정. 파일 업로드에 사용. 얘만 등장하면 서버쪽 프로그래밍이 복잡해진다.                          
                         파일 업로드 사용 시 보안성에 문제가 생길 가능성이 있다. 해커 침투 경로이기도 하다.  
                         이 입력양식이 있으면 반드시 form 태그에는 enctype="multipart/form-data" 가 있어야 한다.
      type="hidden"   -> 화면에 보이지 않고 값만 저장하는 양식 설정.
                         화면에 보이고 싶지 않지만 서버로 보내고싶은 데이터가 있을때 사용한다.
                         검색 화면에서 많이 사용된다.
                         개발자가 조심해야되는 입력양식 중 하나이다.
                         [PK] 값이 저장되기도 한다.
                         <팁>개발과정에서는 type="hidden" 을 type="text" 로 해놓고 개발하고 완료 후 type="hidden" 로 바꾼다.
      ------------------------------------------------------------------------------------------------------
      type="button"   -> [버튼] 양식 설정. 버튼을 클릭하면 벌어지는 일은 개발자가 코딩해야함. 사용자 정의 버튼이다.                        
      type="submit"   -> [서버 전송 버튼] 양식 설정. 같은 폼 소속의 데이터를 서버쪽으로 보내는 전용 버튼이다.                        
      type="reset"    -> [초기화 버튼] 양식 설정. 폼 소속의 입력양식 데이터를 초기화 하는 전용 버튼이다.       
      ------------------------------------------------------------------------------------------------------
      name            -> 입력 양식 이름을 지정. 주로 [오라클]의 [컬럼명]과 일치 시킨다.       
      value           -> 입력 또는 선택한 값이 저장되는 속성. 서버의 웹프로그래밍 언어로 전달되고 최종 DB의 테이블에 저장.
                         <주의>type="button", type="submit", type="reset" 의 경우 버튼에 삽입되는 단순 문자열 설정이고
                         서버의 웹프로그래밍 언어로 전달되지 않는다.
      ------------------------------------------------------------------------------------------------------
      size            -> 입력 양식의 좌우 길이를 설정.
                         <주의> type="text", type="password" 에서는 좌우 길이지만 select 태그안에서는 펼쳐놓는 목록개수를 말한다.
      maxlength       -> 키보드로 입력 가능한 문자열의 개수 설정.
                         <주의> type="text", type="password" 에만 설정 가능.
                         <주의> 입력한 데이터가 한글일 경우 브라우저마다 maxlength속성이 안 먹힐수도 있다.
                                이럴 경우 입력 가능한 문자열의 개수는 [자바스크립트(javascript)]서 체크한다.
      ------------------------------------------------------------------------------------------------------                          
      checked         -> type="checkbox", type="radio" 인 양식에서 삽입되어 선택(=체크) 하게 만듦.
                         <예><input type="checkbox" name="lunar_or_solar" value="lunar" checked>양력
  ---------------------------
  select
  ---------------------------
    ▶1개 이상의 목록 중 하나를 선택하는 [목록상자]를 출력하는 태그다.  
    ▶목록을 감추고 클릭하면 목록을 볼 수 있어 공간을 적게 차지하는 장점이 있다.
    ---------------------------
    ▶select 태그의 속성.
    ---------------------------
      name     -> 입력 양식 이름 지정 속성이다. 주로 [오라클]의 [컬럼명]과 일치 시킨다.
      size     -> select 태그안에서는 펼쳐 놓는 목록개수를 말한다.
      multiple -> 다중 선택 여부를 지정. multiple 설정 시 ctl 키 누르고 다중 선택 가능.
                  주로 관리자 모드에서 사용.
  ---------------------------
  option
  ---------------------------
    ▶select 태그 내부에서 사용되어 [목록상자] 내부의 목록을 출력하는 태그다.
    ---------------------------
    ▶option 태그의 속성.
    ---------------------------
      value           -> DB에 저장될 실제값 저장. <예><option value="1">JSP</option> 에서의 1을 의미
      selected        -> 목록을 선택하여 하게함. <예><option value="JSP" selected>JSP</option>
  ---------------------------
  textarea
  ---------------------------
    ▶다중 행으로 문자열을 입력 할 수 있는 [입력 양식]을 출력하는 태그다.
    ---------------------------
    textarea 태그의 속성.
    ---------------------------
      name     -> 입력 양식 이름 지정 속성이다. 주로 [오라클]의 [컬럼명]과 일치 시킨다.
      rows     -> 보이는 행의 개수 설정
      cols     -> 보이는 열의 개수 설정
```

### <17> HTML에서 이벤트가 뭡니까?

```
● 웹브라우저에서 발생하는 모든 움직임을 발한다.
● 이벤트는 주로 유저에 의해 발생되고, 이벤트는 웹서버에게 일종의 명령을 내리는 것과 동일하다.
  즉, 이벤트는 웹브라우저와 웹서버간의 소통 역할을 한다.
● 웹브라우저에서 발생하는 이벤트는 웹브라우저가 감지하고 있다.  
----------------------------------------
● 웹브라우저가 감지할 수 있는 이벤트의 종류
----------------------------------------
  ---------------------------------------------------------------------------------
  click 이벤트        ->  마우스 왼쪽 버튼을 클릭하는 행위이다. 제일 많이 사용하는 이벤트이다.
  ---------------------------------------------------------------------------------
  mouseover 이벤트    ->  글씨나 이미지에 마우스를 갖다 대는 행위이다.
  ---------------------------------------------------------------------------------
  mouseout 이벤트     ->  글씨나 이미지에 마우스를 갖다 댄 후 떼는 행위이다.
  ---------------------------------------------------------------------------------
  mousedown 이벤트    ->  마우스에 달려 있는 임의 마우스 버튼을 클릭하는 행위이다.
  ---------------------------------------------------------------------------------
  mousedragged 이벤트 -> 마우스를 누른 상태에서 움직이는 행위이다.
  ---------------------------------------------------------------------------------
  keydown 이벤트      -> 키보드를 누르는 행위이다. 키보드를 누르고 있을 때 한번만 이벤트가 발생한다.
  ---------------------------------------------------------------------------------
  keypress 이벤트     -> 키보드를 누르는 행위이다. 키보드를 누르고 있을때 계속 이벤트가 발생한다.
  ---------------------------------------------------------------------------------
  keyup 이벤트        -> 키보드를 누른 후 떼는 행위이다.
  ---------------------------------------------------------------------------------
  select 이벤트       -> 문자열을 드래그해서 선택하는 행위이다.
  ---------------------------------------------------------------------------------
  change 이벤트       -> select 태그라면 목록을 바꿔 선택하는 행위이다.

                        <input type="checkbox" ~> 또는 <input type="radio" ~> 에서는
                        체크가 들어가거나 체크가 빠지는 이벤트이다.

                        <input type="text" ~> 또는 <input type="password" ~> 또는 <textarea ~> 에서는
                        입력 내용이 기존과 다른 후 커서가 빠져나올 때 발생하는 이벤트이다.
  ---------------------------------------------------------------------------------                        
  load 이벤트         -> <body> 태그 안의 모든 내용을 읽어 화면에 출력(=로딩) 한 후 이벤트이다.
  ---------------------------------------------------------------------------------
  unload 이벤트       -> 현재 보여지는 웹페이지가 웹브라우저 화면에서 사라질때 발생하는 이벤트
  ---------------------------------------------------------------------------------
  기타 등등 이벤트  
```
