### # <리액트 문제14> 리액트의 안에서 사용되는 자바스크립트에서 함수 형태를 정리한다면? 
> 화살표함수.  

```
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
<리액트 문제14> 리액트의 안에서 사용되는 자바스크립트에서 함수 형태를 정리한다면? 
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
    -----------------------------------------------------------------------
    일반함수
    -----------------------------------------------------------------------
        function 함수명( [매개변수명] ){
            실행구문;
            [return 리턴값;]    
        }
    -----------------------------------------------------------------------
    클래스 컴포넌트 내부에 선언된 메소드 모양 함수 
    -----------------------------------------------------------------------
        함수명( [매개변수명] ){
            실행구문;
            [return 리턴값;]    
        }
    -----------------------------------------------------------------------
    변수 안의 익명함수
    -----------------------------------------------------------------------
        변수명 = function( [매개변수명] ){
            실행구문;

    -----------------------------------------------------------------------
    변수 안의 화살표 함수 형식1 (매개변수 0개이상, 실행구문 있고 리턴값이 0~1개 이상인 경우)
    -----------------------------------------------------------------------
        변수명 = ( [매개변수명] ) => {

            [return 리턴값;]    
        }
    -----------------------------------------------------------------------
    변수 안의 화살표 함수 형식2 (매개변수 1개, 실행구문 있고 리턴값이 0~1개 이상인 경우)
    -----------------------------------------------------------------------
        변수명 = 매개변수명 => {
            실행구문;
            [return 리턴값;]    
        }    
    -----------------------------------------------------------------------
    변수 안의 화살표 함수 형식3 (매개변수 0개 이상, 실행구문 없고, 리턴값만 있을 경우)
    -----------------------------------------------------------------------
        변수명 = ( [매개변수명] ) => 객체아닌리턴값;
        변수명 = ( [매개변수명] ) => (객체리턴값);
    -----------------------------------------------------------------------
    변수 안의 화살표 함수 형식4 (매개변수 1개, 실행구문 없고, 리턴값만 있을 경우)  
    -----------------------------------------------------------------------
        변수명 = 매개변수명 => 객체아닌리턴값;
        변수명 = 매개변수명 => (객체리턴값);

    -----------------------------------------------------------------------
    <주의> 화살표 함수 안에서 등장하는 this.속성변수
    -----------------------------------------------------------------------


    -----------------------------------------------------------------------
    <주의> 화살표 함수가 아닌 함수 안에서 등장하는 this.속성변수
    -----------------------------------------------------------------------
        this.속성변수 내포하는 바로 위 객체의 속성변수를 가르킨다. 
```
> 화살표 함수에서, 블럭이 없으면, 객체메위주아닌값(실수,문자,boolean값)이 있다면, 이 값은 리턴이 없어도 무조건 리턴이 된다.  
> 중괄호가 없으면, 그 데이터가 리턴된다.  
> 그 값이 메위주면 `()`로 감싸주어야 한다.  

> 매개변수를 끌어안는 괄호도 없는 경우,  
> 매개변수명이 하나 나오고,  화살표 오른쪽에 값을 리턴하는경우는 둥근 괄호 생략 가능.  

- 화살표 함수와 화살표 아닌함수의 차이점?   

  ```js
  function Xxx(){
    this.name = "흰둥이"
    return{
      name:'검둥이'
      ,bark:function(){
        alert( this.name ); // 검둥이 출력
      }
    }
  }
  ```
  > 익명함수
  > 함수의 모양을 띠고 있으나,  
  > 객체화 가능.  
  > 클래스 함수라고도 함.  
  > alert 안의 this 는 익명함수 안에서 this.name 은 '검둥이'를 가리키는 것이다.  

  ```js
  function Xxx(){
    this.name = "흰둥이"
    return{
      name:'검둥이'
      ,bark:()=>{
        alert( this.name ); // 흰둥이 출력
      }
    }
  }
  ```
  > bark 오른쪽에 `=>` 화살표함수가 나오면,  
  > this.name 은 최종 바깥쪽의 '흰둥이'를 지칭한다.  


  ```js
  function twice(value){
      return value*2;
  }

  // 위 아래는 같은값이 계산되어 리턴되는 함수들이다.  

  const twice = (value) => value*2;
  ```
  > 화살표 함수로 간단하게 표현 할 수 있다.  
  > 블럭이 없으면 그 데이터를 바로 리턴한다는 뜻이다.
