<4>트랜잭션(Transaction) 이란 뭡니까? (입.기.실)

```
column  
Transaction
Schema  

영어 표기 주의!  
```

Transaction은 무조건 걸리는게 아니라 내 목적에 따라 걸어주는 것.  
필요하면 내가 설정하는것. 의도를 가지고 설정.  

모두 취소, 모두 들어가던가. 2가지 결과만 진행!  

ex) 2개 이상 UPDATE 진행시..  중간에 오류 나면 다 취소하는것.  

실무적으로는 입력, 수정, 삭제 할때 걸어줌.

Transaction이 안걸리고 오류나면 -> 초기로도 돌릴수 없고 애매하게 들어간 상태.

Transaction이 걸린 작업은 마지막에 COMMIT이나 ROLLBACK이 들어가야한다.

Transaction 걸고 원하는대로 다들어가고 COMMIT 때리기 전에 컴퓨터 꺼지면 들어가지 않는다. 다 취소.

COMMIT 때리기전까진 가상 작업이다.

실제적으로 Transaction 은 입력, 수정, 삭제 때 이용.// 검색은 아님!

SELECT는 Transaction걸 필요가 없다.
